# 生产实习项目 — 客户数据管理系统

## 对话记录（与deepseek-v4-flash）

### 初始情况
- 用户是北航计算机学院学生，需要完成生产实习
- 实习单位：安阳博易通网络有限公司（堂哥的公司）
- 公司以销售业务为主，缺乏计算机相关项目
- 用户获得了公司的客户数据（柳州 & 鄂尔多斯 Excel 表格）
- 最初想编造实习经历，后决定改为用真实数据做真实项目

### 确认的技术方向
- 做客户数据管理系统（Web 应用），包含数据分析和可视化
- 不上单纯 demo，而是可以真正上线使用的系统
- 技术选型待定，倾向 Python/JavaScript 全栈方案
- 系统应对堂哥公司的销售团队有实际使用价值

### 数据概况
- `DataTables/柳州.xlsx`：165 行，字段包括日期、区域、地址、板块、面积、电话、到期、销售、备注
- `DataTables/鄂尔多斯.xlsx`：237 行，字段包括日期、区域、地址、行业/板块、面积、电话、到期日期、销售名、账号名
- 两个城市共约 400+ 条客户记录

### 项目方案定稿（2026-05-11）
- 项目名称：博易通客户数据管理系统
- 技术栈：Flask + SQLite + Chart.js
- 部署方案：Railway 免费部署
- 决定做真正可用的系统并上线，而非本地 demo

### 对原方案与行业标准的差距分析
- 原方案（Flask + Jinja2 + SQLite）与大厂标准流程存在差距
- 主要差距：非前后端分离、无测试、无 CI/CD、SQLite 非生产级数据库

### 技术方案升级确定
- 决定升级为符合大厂标准的技术架构：
  - **后端**：Spring Boot 3.x + Spring Data JPA + PostgreSQL
  - **前端**：React (Vite) + Tailwind CSS
  - **API 风格**：RESTful API
  - **测试**：JUnit 5 + Mockito（后端）
  - **CI/CD**：GitHub Actions
  - **API 文档**：SpringDoc OpenAPI (Swagger)

### 架构定稿
- **后端分层**：Controller → Service(接口+实现) → Repository(JPA) → Entity
- **前端分层**：pages/ → components/ → services/api.js
- **数据流**：用户操作 → React state → axios 调 API → Spring Boot 接口 → PostgreSQL
- **用户 Java 水平一般**，但愿意在 Claude 辅助下用 Spring Boot 开发
