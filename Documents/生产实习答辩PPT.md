## 第2页：项目背景与需求

**公司背景**：安阳博易通网络有限公司主营店铺转让/出租中介服务，销售团队覆盖多个城市。

**原有痛点**：
- 客户信息分散在 Excel 表格中，两个城市字段不一致
- 查询困难，无法多条件筛选
- 没有权限控制，所有人可查看所有数据
- 销售流程无法跟踪（商机状态、跟进记录）

**目标**：替代 Excel，搭建可实际使用的客户数据管理系统

## 第3页：系统总体架构

**架构图（建议用图，文字描述如下）：**

```
┌──────────────────────────────────────────┐
│      前端 (React + Ant Design)            │
│  仪表盘 / 客户列表 / 商机 / 合同 / AI     │
├──────────────────────────────────────────┤
│         RESTful API (JSON + JWT)          │
├──────────────────────────────────────────┤
│    后端 Spring Boot 3.x (Java 17)         │
│  Controller → Service → Repository(JPA)  │
│  Spring Security + JWT 认证               │
├──────────────────────────────────────────┤
│         PostgreSQL / H2 数据库             │
└──────────────────────────────────────────┘
```

**技术栈**：
| 后端 | Spring Boot 3.x / JPA / Spring Security / JWT |
| 前端 | React / Ant Design / Chart.js |
| 数据库 | PostgreSQL / H2 |
| 部署 | Docker / Railway |
| AI | DeepSeek V3.2 大语言模型 |

## 第4页：核心功能模块与成果

**功能模块总览**：

| 模块 | 功能 |
|------|------|
| 客户管理 | 多条件筛选、状态流转（线索→跟进→成交）、Excel导入导出 |
| 商机管理 | 6阶段推进、赢单率自动计算 |
| 合同管理 | 自动编号、状态流转、回款计划 |
| 产品管理 | 产品CRUD |
| 数据统计 | 仪表盘、4种可视化图表、销售漏斗 |
| 用户权限 | JWT认证、业务员数据隔离 |
| AI 助手 | 智能对话、跟进记录自动生成 |

**项目规模**：
- 后端：73个Java文件 / ~3,800行代码
- 前端：30个文件 / ~2,500行代码
- 客户数据：2,560条，覆盖12个城市

## 第5页：亮点功能 — AI 智能助手

**应用场景**：
1. 浮动聊天窗口：用户可直接与AI对话，查询客户数据
2. 跟进记录自动生成：在客户详情页，AI根据客户信息自动生成跟进文案

**技术实现**：
```
用户在聊天框提问 → 后端拼接客户数据上下文 → 调用 DeepSeek API → 返回专业回答
```

**效果**：业务员撰写跟进记录的效率大幅提升，文案质量也有保障。
