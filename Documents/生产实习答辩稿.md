# 生产实习答辩稿（5分钟版）

**总时长**：约5分钟
**语速**：正常语速，不要太快

---

## 一、开场（30秒）

各位老师好。我在安阳博易通网络有限公司完成了为期十周的生产实习，实习岗位是软件开发，课题是《博易通客户数据管理系统的设计与开发》。

安阳博易通是一家专注于店铺转让与出租中介服务的公司，之前一直用 Excel 管理客户信息，随着业务扩大，逐渐暴露出查询困难、权限混乱、协作低效等问题。

我的任务是开发一套替代 Excel 的客户数据管理系统。

---

## 二、系统架构（40秒）

整个系统采用前后端分离架构。后端用 Spring Boot 3.x 加 Spring Data JPA，前端用 React 加 Ant Design 组件库，数据库用 PostgreSQL，用户认证采用 JWT 方案。

前后端通过 RESTful API 通信，项目通过 Docker 容器化部署到 Railway 平台。代码托管在 GitHub，配置了 CI 流水线，每次推送自动构建和测试。

---

## 三、核心功能（1分30秒）

系统主要包含七个功能模块。

客户管理是核心，支持多条件筛选和关键词搜索，业务员只能看到分配给自己的客户数据。客户状态可以从新线索流转到跟进中、洽谈中、已成交、已流失。

商机管理将销售过程分为六个阶段，从意向客户到方案报价、报价确认、商务谈判、赢单和输单，每个阶段自动计算赢单率。

合同管理支持合同创建、自动编号和状态流转，关联回款计划。

此外还有产品管理、数据仪表盘和公告中心等功能模块。

项目规模方面，后端有 73 个 Java 文件约三千八百行代码，前端有 30 个文件约两千五百行代码，系统管理着两千五百多条覆盖十二个城市的客户数据。

---

## 四、亮点功能：AI助手（50秒）

这里重点介绍 AI 助手功能。系统接入了 DeepSeek V3.2 大语言模型，实现了两个应用场景。

一是全局浮动聊天窗口，用户可以直接在系统内和 AI 对话，AI 能根据系统的实时数据回答业务问题。二是跟进记录自动生成，在客户详情页点击按钮，AI 会根据客户信息自动生成专业的跟进文案，业务员可以直接采用或稍加修改。

这个功能大大提高了业务员撰写跟进记录的效率，也是系统的特色亮点。

---

## 五、测试与部署（30秒）

测试方面，我编写了 95 个后端单元测试用例，覆盖了客户管理、CRM 模块、AI 助手、个人中心、操作日志、通知等所有核心功能等所有核心模块，全部测试通过。部署方面，系统已经通过 Docker 容器化部署到 Railway 平台，部署到了 Railway 平台，可以通过域名在线访问。

---

## 六、总结（30秒）

通过这次实习，我完整经历了一个真实项目从需求分析、系统设计到开发测试、部署上线的全过程。不仅巩固了 Spring Boot 和 React 的技术能力，更重要的是学会了如何从业务需求出发做技术选型，体会了软件工程规范在真实项目中的价值。
